apps.analytics.views.collect_stats_view(request)[source]

Представлення для збору статистики та генерації графіків аналітики.

Обробляє POST-запити для створення графіків аналітики через функцію get_analytics(). При успішному створенні оновлює кеш та повертає JSON-відповідь.

Параметри:

request – HTTP-запит

Повертає:

JSON-відповідь зі статусом операції:

  • success: Графіки успішно створені

  • error: Виникла помилка при створенні або невірний тип запиту

Тип повернення:

JsonResponse

Context Processors

Контекстний процесор для додавання графіків аналітики до контексту шаблонів.

Отримує список графіків з кешу або з бази даних, якщо кеш порожній. Кешує результат без обмеження за часом.

Параметри:

request – HTTP-запит

Повертає:

Словник з ключем „graphs“, що містить список об’єктів AnalyticGraph

Тип повернення:

dict

Приклади використання

Генерація аналітики

from ml.analytics.analytics_creator import get_analytics

# Створення всіх графіків
get_analytics()

Отримання графіків

from apps.analytics.models import AnalyticGraph

# Всі графіки
graphs = AnalyticGraph.objects.all()

# Конкретний графік
graph = AnalyticGraph.objects.get(name='loan_distribution')

# Використання властивостей
print(graph.templates_name)  # "Loan Distribution"
print(graph.image_url)       # "/media/loan_analysis_plots/01_loan_distribution.png"

AJAX запит на генерацію

fetch('/analytics/collect-stats/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json'
    }
})
.then(response => response.json())
.then(data => {
    if (data.status === 'success') {
        console.log('Графіки створено!');
        location.reload();
    }
});
